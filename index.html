HTML

  • Anything in the head element is not displayed to the user
  • The head element contains information about the webpage
  • The body element represents the visible content shown to the user
  • Metadata elements are stored within the head element
  • Elements that are nested inside other elements are "child elements"
  • The element that holds all of the nested elements is called the "parent element"
  • Section elements need to put at the front and back, with a "/" included in the back section element

CSS

  • How to link a .css file to an HTML file:
    • link rel="stylesheet" href="./assets/style.css"
      • link: creates a link to an external file
      • rel: specifies the relationship between the current document and the linked documnet
      • href: the url for the linked file
  • CSS Rule:
    • div {color: blue; text-align: justify;}
      • div = sector
      • color, text-align = property
      • blue, justify = value
      • Declaration block = everything inside {}
  • Using a "*" makes any folling changes apply to the entire code
  • You can combine elements: using h1, h2{style effects here}
  • You can also change picture dimensions with img {style in here}
  • A margin indicates how much space we want around the outside of an element
  • A padding indicates how much space we want around the content inside an element
  • Adding a "." before anything indicates that it is a class
  • We styled all class elements using .card in style.css

Git

  • Every git commands starts with "git"
  • git status: checks what branch we are currently working in
  • git checkout -b branchname: creates a new branch and switches to it
    • checkout: move to the working branch
    • -b: creates a new branch
    • branchname: whatever you want to name the branch
  • git clone SSH value: will clone the current version of the file off github and put it in your computer
  • git pull origin main
    • pull: get the version from github onto your local machine
    • origin:
  • git add -A
    • add: adds our changes to a staging area
    • -A: indicates that we want to add all changes
  • Before we push our code to github, we want to make sure our local branch is still in sync with the base branch:
    • git pull origin main
  • git push branchname
  • When reworking in a branch, push changes to the branch that the changes are being made in

JavaScript

  • Javascript allows users to interact with the website
  • Variables are created by using "var name = ..."
    • Var can store more than just numbers, but also strings
  • Use chromeDev Tools: Fn + F12
  • Arrays, [], can store differnt types of variables:
    • "Strings"
    • numbers
    • boolean
  • Items in an arry can be called by arrayname[numbergoeshere]
    • Arrays start at 0
  • Arrays, functions, and variables are all case sensative
  • A variable is a named container that allows us to store data in our code
  • Control flow is the order in which a computer executes code in a script
    • JavaScript runs from top to bottom, so variables must be created before being called
  • for loop:
    • for(x; y; z)
  • Functions always end in a ()
    • parameters will go inside the ()
